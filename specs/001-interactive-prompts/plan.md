# Implementation Plan: Interactive CLI Prompts

**Branch**: `001-interactive-prompts` | **Date**: 2026-03-03 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-interactive-prompts/spec.md`

## Summary

Add InquirerPy-powered interactive prompts to lola's CLI so that omitting `module_name`
or `-a/--assistant` triggers a keyboard-navigable picker rather than an error or silent
all-assistant default. A new `src/lola/prompts.py` module centralises prompt logic and
TTY detection; existing command code is adjusted at the argument-gathering layer only.
Non-interactive (CI/piped) invocations preserve current behaviour throughout.

## Technical Context

**Language/Version**: Python 3.13 (modern type hints; `list[str]`, `str | None`, etc.)
**Primary Dependencies**: click (CLI), rich (console output), InquirerPy 0.3.4 (new runtime dep), prompt_toolkit (transitive)
**Storage**: N/A — no new persistent storage; existing installation registry (JSON) unchanged
**Testing**: pytest, Click CliRunner, `unittest.mock.patch`
**Target Platform**: macOS, Linux; Windows Terminal (ANSI-capable)
**Project Type**: CLI tool
**Performance Goals**: Prompt renders in < 200 ms; no perceptible latency added to non-interactive paths
**Constraints**: Backward-compatible — all explicit-flag invocations must behave identically to today
**Scale/Scope**: 4 assistants in TARGETS; prompt lists bounded by number of registered/installed modules

## Implementation Phases

### Phase A — Foundation (prompts.py + dependency)

1. Add `InquirerPy>=0.3.4` to `pyproject.toml` dependencies; run `uv sync`.
2. Create `src/lola/prompts.py` with four functions:
   - `is_interactive() -> bool` — `sys.stdin.isatty()`
   - `select_assistants(available: list[str]) -> list[str]` — checkbox prompt
   - `select_module(modules: list[str]) -> str | None` — select prompt
   - `select_marketplace(matches: list[tuple[dict, str]]) -> str | None` — select with descriptions
3. Add unit tests in `tests/test_prompts.py` (mock `inquirer.*`, mock `sys.stdin.isatty`).

### Phase B — `install_cmd` Changes

1. Make `module_name` argument optional (`required=False, default=None`).
2. At the top of `install_cmd`, before module resolution:
   - If `module_name is None` and non-interactive → error + exit(1).
   - If `module_name is None` and interactive → call `select_module(registered_names)`.
3. At the assistant-determination line (currently line 769):
   - Replace `assistants_to_install = [assistant] if assistant else list(TARGETS.keys())`
   - With: if `assistant` provided → use it; elif interactive → call `select_assistants`; else → all (non-interactive default).
4. Update `install_cmd` docstring and help text.
5. Add/update tests in `test_cli_install.py` covering the new paths.

### Phase C — `uninstall_cmd` Changes

1. Make `module_name` argument optional (`required=False, default=None`).
2. At the top of `uninstall_cmd`, before registry lookup:
   - If `module_name is None` and non-interactive → error + exit(1).
   - If `module_name is None` and interactive → gather all installed module names → `select_module`.
3. Update docstring and help text.
4. Add/update tests.

### Phase D — Marketplace Conflict UI

1. Refactor `MarketplaceRegistry.select_marketplace()` in `market/manager.py`:
   - Remove the numbered-list Rich printing and `click.prompt`.
   - Delegate selection to `prompts.select_marketplace(matches)`.
   - Keep the early-return for `len(matches) == 1` (unchanged).
2. Add tests for non-interactive marketplace conflict scenario (expect first match to be returned or user message).

### Phase E — Verification + Documentation

1. Verify **FR-010** exists in `spec.md` (already added during planning).
2. Verify `CLAUDE.md` includes InquirerPy in Active Technologies for `001-interactive-prompts` (already added during planning).
3. Run full test suite (`pytest --tb=short`) and ruff (`ruff check src tests`).

## Key Decisions Summary

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Prompt library | InquirerPy 0.3.4 | Purpose-built for CLI; checkbox + select; no full-screen TUI needed |
| TTY detection | `sys.stdin.isatty()` | Standard POSIX; reliable across macOS/Linux/WSL2 |
| Non-interactive install default | All assistants (unchanged) | Backward-compatible; breaking change avoided |
| Non-interactive uninstall default | Error with clear message | No safe default; explicit is correct |
| New module placement | `src/lola/prompts.py` | Consistent with other lola helper modules |
| Test strategy | Mock at import boundary | CliRunner uses non-TTY stdin; InquirerPy needs real TTY |

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| InquirerPy raises exception on non-TTY | `is_interactive()` check precedes every prompt call |
| Windows compatibility | InquirerPy 0.3.x uses `prompt_toolkit` which supports Windows Terminal |
| Marketplace `select_marketplace` test breakage | Refactor is purely UI; existing test mocks targeting `click.prompt` updated |
| `module_name` now optional breaks existing tests | Click's CliRunner still accepts positional args; only tests that rely on missing-arg error output change |
